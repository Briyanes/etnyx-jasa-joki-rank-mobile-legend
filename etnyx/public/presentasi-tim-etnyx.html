💳 Sistem Pembayaran

Transfer Manual

11 Metode Pembayaran:

  • 🏦 Bank: BCA, BRI, BNI, Mandiri, Jago
  • 📱 E-Wallet: DANA, GoPay, OVO, ShopeePay, LinkAja
  • 📷 QRIS (scan barcode)

Flow:

  • Customer transfer → upload bukti + nama pengirim
  • Admin cek di Telegram → buka Dashboard
  • Verify: nominal, nama, tanggal
  • Approve → auto notif WA customer

Midtrans Otomatis

Jika server key dikonfigurasi:

  • QRIS, Virtual Account, GoPay
  • ShopeePay, Kartu Kredit/Debit

Flow:

  • Customer pilih Midtrans → Snap popup
  • Bayar → webhook auto-confirm
  • Order otomatis: paid + confirmed
  • Notif Telegram + WA langsung

✅ Webhook: amount verify + idempotency + signature check

👑 SOP Admin — Tugas Harian

🌅 Pagi (08-10)

  • Cek Telegram (order baru, bukti transfer)
  • Dashboard → Orders → filter pending
  • Verifikasi bukti transfer → Approve
  • Pastikan order confirmed sudah di-assign

☀️ Siang (12-14)

  • Monitor in_progress (cek % progress)
  • Follow-up pending > 6 jam via WA
  • Cek KPI di Overview
  • Review complaints / worker reports

🌙 Malam (20-22)

  • Verify submissions + screenshots
  • Kirim "Minta Review" ke completed 24+ jam
  • Review Ads (ROAS/CPA)
  • Cek P&L, proses payout jika jadwal

✅ DO's

  • Verify bukti transfer (nominal, nama, tanggal)
  • Confirm payment < 1 jam saat jam kerja
  • Follow-up pending > 6 jam via WA template
  • Approve/resolve reviews dalam 24 jam
  • Bayar komisi bi-weekly tepat waktu

❌ DON'Ts

  • Approve tanpa lihat bukti transfer
  • Biarkan order pending > 24 jam
  • Cancel order tanpa konfirmasi customer
  • Share API keys / ENCRYPTION_KEY
  • Hard-delete order (cancel saja)

🔵 SOP Lead — Koordinator

🌅 Pagi (08-10)

  • Login Lead Dashboard
  • Cek Telegram → order baru
  • Distribusi order ke worker available
  • Cek workload distribution

☀️ Siang (12-14)

  • Monitor progress in_progress
  • Follow-up worker lambat via Telegram
  • Kirim WA progress ke customer jika perlu

🌙 Malam (20-22)

  • Verify completed (submissions + screenshot)
  • Pastikan customer terima WA completion
  • Brief worker untuk order urgent/express

✅ Aturan Lead

  • Assign ASAP setelah confirmed (< 30 menit)
  • Distribusi order merata antar worker
  • Max 3 order aktif per worker
  • Gunakan Bulk Assign untuk banyak order
  • Cek workload sebelum assign

⚠️ Eskalasi ke Admin jika:

  • Worker non-responsive > 2 jam
  • Customer complaint serius
  • Order stuck > 24 jam
  • Laporan misconduct worker
  • Masalah teknis / payment

🟢 SOP Worker — Booster

#LangkahDetail
1Login/admin → auto ke Worker Dashboard
2Lihat Order3 kategori: Menunggu, Aktif, Selesai
3Cek CredentialsLihat akun (encrypted AES-256, hanya worker assigned)
4Mulai OrderKlik "Mulai" → status: in_progress, WA ke customer
5Update ProgressSlider 0-100% + rank saat ini (customer bisa track)
6Submit HasilStars, MVP, Savage, Maniac, Wins, Duration + screenshot WAJIB
7SelesaiKlik "Selesai" → komisi 60% auto → WA ke customer + review link
8Logout MLLogout dari akun ML customer setelah selesai

✅ Aturan

  • Cek credentials login OK sebelum mulai
  • Update progress % setiap beberapa match
  • Upload screenshot setiap sesi (wajib)
  • Report masalah akun segera ke Lead/Admin

🚫 Dilarang

  • Login dari device berbeda (risiko ban akun)
  • Lihat order worker lain
  • Edit submission setelah 30 menit
  • Skip upload screenshot

🔒 Keamanan Sistem

🔐 Enkripsi & Auth

  • AES-256-GCM untuk credentials customer
  • Format: iv:authTag:encrypted
  • Hanya worker assigned yang bisa decrypt
  • Password reset: token 1x pakai, 1 jam expired
  • JWT-based session management

🚨 Rate Limiting

  • General API: 100 req/min per IP
  • Auth endpoints: 10 req/5min per IP
  • Sliding window (in-memory)
  • Middleware-level blocking

🛡️ Database Security

  • Row Level Security (RLS) on semua tabel core
  • Service role bypass untuk API routes
  • Public read hanya whitelist tertentu
  • Admin audit log (semua aksi tercatat)

✅ Audit & Testing

  • 38+ unit tests (Vitest)
  • Playwright E2E setup
  • 19 security fixes (mass assignment, IDOR, dll)
  • Webhook idempotency + amount verification
  • Health check endpoint (Supabase, Midtrans, Telegram, Email)
